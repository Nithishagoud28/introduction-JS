JAVA SCRIPT

INTRODUCTION:

  • JAVA SCRIPT: It is a collection of programs to receive request & provides response by applying some business logics.
  • Application: Collection of programs to build any software.
  • Program: Set of instructions to do any task.
  • From Frontend/UI we can recieve any request. Only static application(html,css.)
  • Dynamic Application: We can change by request(html css js)raised by user webpage.by using js we can make an application dynamic.
  • Standalone Application: Application works without any internet. Ex: Calculator , Gallery.
  • We can perform from validation using js.
  • static Application: HTML,CSS,JS
  • Dynamic Application: This application runs based on request & response Model(html,css,js,Bootstrap.)
  • Enduser will raise the request through Frontend/UI.
  • We need one processor/server for providing response based on request.
  • Server Should maintain at Backend.
  • API: Application Programming interface, it will make communication with two systems.
  • Js runs only with browser.
  • With the help of Node.Js we can run anywhere i.e (out of browser)

ENGINES:

  • Chorme-V8 Engine.
  • Firefox: SPIDERMONKEY ENGINE.
  • Edge: CHAKRA ENGINE (old version replaced now by v8.)
  • Safari: NITRO(js core)
  • This engine are nothing but a piece of code.
  • V8 engine is developed by combo of c++,js,python,type script.
  • Because of v8 engine in chorme we can run js in chrome.

3-Tier Architecture-3 Phases.

  • Frontend Presentation layer: HTML,CSS JS.
  • Backend Application layer: Node js,python,SQL Java
  • Data-base: User interface layer(to store the data.)
  • Frontend and Backend are connected Through API.

WHY JS?.

  • For making application dynamic.
  • For form validation.
  • Raising request from Frontend and Providing response to backend.
  • In Frontend- Js will act as a Scripting Language.
  • In Backend-Js will act as a Programming language.
  • Js can run only in browser with the help of js engines.
  • If we want to run js out of the brower then node js will help us.
  • Node js can create a runtime environment for js with the help of v8 engine.
  • Any application will works on request and response Model.
  • So, it Should have 3 phases-Frontend,backend and data-base we can called 3-Tier Architecture
  • With the help of js we can interact with web content.
